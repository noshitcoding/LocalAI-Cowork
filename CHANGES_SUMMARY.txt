IMPLEMENTATION SUMMARY: Per-Chat/Per-Task Permission Mode and Allowed Directories
================================================================================

PROBLEM SOLVED:
--------------
Previously, when users set folder permissions or permission mode (bypass, strict, etc.)
for a specific chat or task, it was applied globally to all chats and tasks. This was
because permissionMode and allowedDirectories were stored in the global engineStore
config.

SOLUTION IMPLEMENTED:
--------------------
Added per-chat and per-task permission configuration that allows each chat thread to
have its own permission mode and allowed directories list.

KEY CHANGES:
-----------

1. Type Definitions (chatStore.ts, taskStore.ts)
   - Added PermissionConfig type: { mode: PermissionMode, allowedDirectories: string[] }
   - Added permissionConfig?: PermissionConfig to ChatThread and Task types

2. Engine Store (engineStore.ts)
   - sendMessage() now accepts permissionConfig parameter
   - _initEngine() now accepts permissionConfig parameter  
   - buildChatEngineConfig() uses permissionConfig or falls back to global settings
   - Config resolution: permissionConfig?.mode ?? config.permissionMode

3. Query Engine (queryEngine.ts)
   - Added allowedDirectories?: string[] to EngineConfig
   - Updated buildToolContext() to include allowedDirectories in permission context
   - Permission checks now respect per-chat allowed directories

4. Database (db.rs)
   - Migration v16: Added permission_config_json column to chat_threads table
   - Updated list_threads() to include permission_config_json
   - Updated insert_thread() to accept permission_config_json
   - Added update_thread_permission_config() method

5. UI Components
   - ChatView.tsx: Added permission config panel (🔒 button)
     * Mode selector: default/plan/bypass/strict
     * Allowed directories: add/remove
     * Save/cancel functionality
   - CoworkView.tsx: Updated to pass thread's permission config
   - WelcomeScreen.tsx: Updated to pass thread's permission config

6. Styling (App.css)
   - Added CSS for permission config panel
   - Positioned as floating panel in top-right

7. Tests (permissionConfig.test.ts)
   - Added unit tests for PermissionConfig type
   - All tests passing

USAGE:
------
1. Open any chat thread
2. Click 🔒 "Berechtigungen" button (top-right)
3. Select permission mode:
   - Standard: Normal tool approval flow
   - Plan-Modus: Planning only, no execution
   - Bypass: No approval prompts, all tools allowed
   - Strikt: All tools require approval
4. Add allowed directories (optional):
   - Type path and click "Hinzufügen"
   - Click × to remove
   - Empty = no restrictions
5. Click "Speichern"
